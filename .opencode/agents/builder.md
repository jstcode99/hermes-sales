## Mandatory project patterns

### Implementation order for a new entity
1. `types/<type>.ts` + enums - generated by Supabase
2. `@modules/<module>`
3. `@modules/<module>/services/<service>.service.ts` — with `getCached*` methods for reads
5. `@modules/<module>/schemas/<schema>.schema.ts` — use base schemas from `common/schemas.ts`
6. `@modules/<module>/mappers/<mapper>.mapper.ts`
8. `@modules/<module>/actions/<action>.action.ts`
7. `@modules/app.modules.ts` — registers modules at the global level
9. `features/<domain>/` — UI components

### Non-negotiable rules
- `modules()` is the only place where services are instantiated
- Supabase clients only from `@lib/supabase` — never directly
- `CACHE_TAGS` from `constants.ts` — never raw strings in `revalidateTag`
- `@/` for all imports — never relative paths
- No `any` in TypeScript
